# Flask MongoEngine CRUD (User)

Lightweight example REST API using Flask and MongoEngine to perform basic CRUD operations for a `User` model.

## Summary

- Simple service exposing endpoints to create and fetch users.
- Uses `mongoengine` (via `flask_mongoengine`) for MongoDB object mapping.
- Minimal, easy-to-read project structure suitable as a starter template.

## Features

- Create a user (POST)
- List all users (GET)
- Get a user by id (GET)

## Requirements

- Python 3.8+
- MongoDB instance (local or hosted)
- Python packages: `Flask`, `flask-mongoengine`, `mongoengine`, `python-dotenv`, `pymongo`

You can install required packages with:

```bash
python -m venv .venv
source .venv/bin/activate    # or `.venv\\Scripts\\activate` on Windows
pip install Flask flask-mongoengine mongoengine python-dotenv pymongo
```

## Configuration

Create a `.env` file in the project root with the MongoDB connection string. The app expects the environment variable `MONGO_URI`.

Example `.env`:

```
MONGO_URI=mongodb://username:password@host:port/database
```

The application loads the env vars via `python-dotenv` in [app/**init**.py](app/__init__.py#L1).

## Run

Start the app with:

```bash
python run.py
```

By default the app binds to `0.0.0.0` (see [run.py](run.py#L1)) and runs with `debug=False`.

## API Endpoints

Base prefix: `/user` (registered in `run.py`)

- POST `/user/add`
  - Description: Create a new user
  - Body (JSON): `{ "name": "Alice" }`
  - Response: plain message `user saved`
  - Handler: `add_user` -> [controllers/userController.py](controllers/userController.py#L1)

- GET `/user/all`
  - Description: Return a list of all users (JSON)
  - Response: JSON array of users
  - Handler: `get_user` -> [controllers/userController.py](controllers/userController.py#L1)

- GET `/user/get?userId=<id>`
  - Description: Fetch a single user by `_id`. If not found, returns 404 and a JSON message.
  - Handler: `get_user_by_id` -> [controllers/userController.py](controllers/userController.py#L1)

Example curl commands:

```bash
# Create user
curl -X POST -H "Content-Type: application/json" -d '{"name":"Alice"}' http://localhost:5000/user/add

# List users
curl http://localhost:5000/user/all

# Get user by id
curl "http://localhost:5000/user/get?userId=<id>"
```

## Data Model

The `User` model is defined in [models/user.py](models/user.py#L1):

- `_id` (string): primary key, UUID string generated by default.
- `name` (string): required.

The model uses `mongoengine.Document` via the `db` instance in [app/**init**.py](app/__init__.py#L1).

## Project Structure

- [run.py](run.py#L1) — application entrypoint (registers blueprints)
- [app/**init**.py](app/__init__.py#L1) — Flask app and MongoEngine initialization
- [routes/userRoutes.py](routes/userRoutes.py#L1) — blueprint and route bindings
- [controllers/userController.py](controllers/userController.py#L1) — request handlers
- [services/userService.py](services/userService.py#L1) — business logic / DB operations
- [models/user.py](models/user.py#L1) — `User` document model
